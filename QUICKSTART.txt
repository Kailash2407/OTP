⏱️  QUICK START (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Install Dependencies
   ➜ Open command prompt in: d:\motor-dashboard\
   ➜ Run: npm install
   ⏱️  Takes about 2 minutes

STEP 2: Add AWS IoT Certificates
   ➜ Copy these 3 files to d:\motor-dashboard\ (root directory):
      • private.key
      • certificate.pem.crt
      • AmazonRootCA1.pem
   ⏱️  Takes 30 seconds

STEP 3: Start the Server
   ➜ Run: npm start
   ➜ Or double-click: setup.bat (Windows automatic setup)
   ⏱️  Takes 10 seconds

STEP 4: Open Dashboard
   ➜ Open browser and go to: http://localhost:3000
   ➜ You should see the motor dashboard!
   ⏱️  instant

STEP 5: Test with MQTT Data
   ➜ Publish this JSON to topic "vehicle/motor/data":
      {
        "speed": 2500,
        "current": 12.5,
        "voltage": 48.0,
        "temperature": 65.5
      }
   ➜ Dashboard updates in real-time! ✨

🚨 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "npm: command not found"
Solution: Install Node.js from https://nodejs.org/

Problem: "Cannot find module" after npm install
Solution: Run "npm install" again or delete node_modules and reinstall

Problem: "Port 3000 in use"
Solution: Use different port: PORT=8080 npm start

Problem: "AWS connection fails"
Solution: Verify certificate files exist and paths are correct in server.js

Problem: Dashboard shows "OFFLINE"
Solution: Check server is running (npm start). Check firewall allows port 3000

Problem: "No data appearing on dashboard"
Solution: Verify MQTT message is published to correct topic with correct format
